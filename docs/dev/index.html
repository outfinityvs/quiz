Developer Documentation

Technical documentation for the Outfinity Quiz engine, architecture, and specifications.

Architecture Overview

The quiz subsite is a static, dependency-free application served from the docs/ directory. The engine is a set of vanilla JavaScript modules that load quiz definitions from JSON files and render interactive experiences in the browser.

Engine Modules

  • engine/quiz-engine.js — State machine, question rendering, scoring, and result computation.
  • engine/banner-presets.js — Central initiative banner presets assigned by key from quiz JSON.
  • engine/result-history.js — Explicit browser-local saved result history, menu integration, Save Result controls, and catalog saved-result indicators.
  • engine/radar-chart.js — SVG radar/polygon chart with configurable axes, multi-overlay support, and presentation-only sizing controls.
  • engine/share-card.js — Canvas-based share card generation (1080×1350) with PNG download.
  • engine/url-codec.js — Base64URL encoding and decoding of quiz results in URL fragments.
  • engine/compare.js — Multi-result overlay comparison with transparency and complementarity analysis.

Quiz Definition Format

Each quiz is defined by four JSON files in its directory under quizzes/:

  • instrument.json — Questions, options, dimensions, scoring weights, branching rules, and version.
  • interpretation.json — Archetypes, result texts, share texts, and recommendations.
  • catalog.json — Metadata for the catalog page (title, description, duration, type, domain).
  • recommendations.json — Optional partner/initiative mappings.

Development Workflow

No build step is required. Serve the docs/ directory with any static file server:

npx serve docs/

Or with Python:

python3 -m http.server 8080 --directory docs/
